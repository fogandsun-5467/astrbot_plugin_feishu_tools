# AstrBot 插件完整开发指南
本文档为AstrBot插件开发全流程官方指南，整合了从环境搭建、基础开发、进阶能力到插件发布的全部内容，适配AstrBot主流版本，关键功能会标注最低版本要求，确保大模型可完整理解并遵循开发逻辑。

## 前置基础要求
1. 具备基础的Python编程能力，熟悉异步编程（async/await）语法
2. 掌握Git、GitHub的基础使用方法
3. 开发者官方交流QQ群：`975206796`

## 一、环境准备与插件项目初始化
### 1.1 获取官方插件模板
1. 打开AstrBot官方插件模板仓库：helloworld
2. 点击仓库右上角的`Use this template`，选择`Create new repository`
3. 填写仓库名称（即插件名），需严格遵循命名规范：
   - 推荐以`astrbot_plugin_`为前缀
   - 不允许包含空格
   - 全部使用小写字母
   - 名称尽量简短易懂
4. 点击`Create repository`完成插件仓库创建

### 1.2 克隆项目到本地
执行以下bash命令，完成AstrBot本体与插件仓库的克隆：
```bash
# 克隆AstrBot项目本体
git clone https://github.com/AstrBotDevs/AstrBot
# 创建插件存放目录
mkdir -p AstrBot/data/plugins
# 进入插件目录
cd AstrBot/data/plugins
# 克隆你创建的插件仓库
git clone 你的插件仓库地址
```
完成后，使用VSCode打开`AstrBot`项目，定位到`data/plugins/你的插件名`目录，即可开始开发。

### 1.3 核心元数据配置
> ⚠️ 必须完成此项配置，AstrBot依赖该文件识别插件元数据
1. 打开插件目录下的`metadata.yaml`文件
2. 按实际需求填写插件的名称、版本、作者、描述等元数据信息

### 1.4 可选配置项
#### 1.4.1 插件Logo设置
在插件根目录下添加`logo.png`文件作为插件Logo，要求：
- 图片长宽比为1:1
- 推荐尺寸为256x256

#### 1.4.2 插件展示名设置
在`metadata.yaml`中修改/新增`display_name`字段，该内容将作为插件在插件市场的展示名称，提升用户可读性。

### 1.5 插件调试方法
1. AstrBot采用运行时注入插件的机制，调试插件必须先启动AstrBot本体
2. 热重载开发：插件代码修改后，进入AstrBot WebUI的插件管理页面，找到对应插件，点击右上角`...`按钮，选择`重载插件`即可使修改生效，无需重启机器人。

### 1.6 依赖管理
1. AstrBot使用pip原生的`requirements.txt`管理插件依赖
2. 若插件使用了第三方Python库，必须在插件根目录创建`requirements.txt`，并写入所有依赖项，避免用户安装时出现`Module Not Found`错误
3. `requirements.txt`格式规范可参考pip官方文档

## 二、插件基础结构与最小实例
### 2.1 插件核心规范
- 插件主文件必须命名为`main.py`
- 插件主类必须继承`Star`基类
- 所有消息处理函数（Handler）必须定义在插件主类中
- Handler前两个参数必须为`self`和`event`
- 必须使用`@register`装饰器注册插件主类

### 2.2 最小可运行插件实例
```python
from astrbot.api.event import filter, AstrMessageEvent, MessageEventResult
from astrbot.api.star import Context, Star, register
from astrbot.api import logger # 官方提供的日志接口

# 插件注册装饰器，参数依次为：插件ID、作者、插件描述、版本号、仓库地址
@register("helloworld", "author", "一个简单的 Hello World 插件", "1.0.0", "repo url")
class MyPlugin(Star):
    def __init__(self, context: Context):
        # 必须调用父类构造方法，context用于与AstrBot Core交互
        super().__init__(context)
        # 插件初始化逻辑写在这里

    # 指令注册装饰器，注册后用户发送/helloworld即可触发该函数
    @filter.command("helloworld")
    async def helloworld(self, event: AstrMessageEvent):
        '''这是一个 hello world 指令'''
        # 函数注释会被解析为指令描述，建议填写
        user_name = event.get_sender_name() # 获取发送者昵称
        message_str = event.message_str # 获取消息纯文本内容
        logger.info("触发hello world指令!") # 打印日志
        # 回复纯文本消息，使用yield返回
        yield event.plain_result(f"Hello, {user_name}!")

    async def terminate(self):
        '''可选实现，插件被卸载/停用时会自动调用'''
        # 插件销毁逻辑，如资源释放、数据保存写在这里
        logger.info("插件已卸载")
```

### 2.3 核心对象说明
| 对象名 | 核心作用 |
|--------|----------|
| `Star` | 插件基类，所有插件必须继承该类 |
| `Context` | 插件与AstrBot Core交互的核心入口，可调用机器人各类API |
| `AstrMessageEvent` | 消息事件对象，封装了消息发送者、内容、会话等全部信息 |
| `filter` | 事件过滤器与指令注册器，用于定义消息触发规则 |

## 三、消息事件与指令系统
AstrBot的所有消息都会被封装为`AstrMessageEvent`对象，通过`filter`模块的装饰器定义触发规则，实现指令、事件监听等功能。
> 必须先导入filter模块：`from astrbot.api.event import filter, AstrMessageEvent`，避免与Python原生filter函数冲突。

### 3.1 核心消息对象
#### 3.1.1 AstrMessageEvent
消息事件顶级对象，包含了消息的全部上下文，核心属性与方法：
- `message_str`：消息的纯文本内容，自动拼接消息链中的所有文本段
- `message_obj`：获取`AstrBotMessage`消息对象
- `unified_msg_origin`：会话唯一标识，用于主动消息发送
- `get_sender_id()`：获取发送者ID
- `get_sender_name()`：获取发送者昵称
- `get_group_id()`：获取群聊ID，私聊返回空
- `get_platform_name()`：获取消息来源平台
- `plain_result(text)`：构建纯文本消息回复
- `chain_result(chain)`：构建富媒体消息链回复
- `send(result)`：主动发送消息（异步方法）
- `stop_event()`：停止事件传播，后续插件/逻辑不会再处理该消息

#### 3.1.2 AstrBotMessage
消息本体对象，存储平台下发的原始消息详情，核心定义：
```python
class AstrBotMessage:
    '''AstrBot 的消息对象'''
    type: MessageType  # 消息类型
    self_id: str  # 机器人ID
    session_id: str  # 会话ID
    message_id: str  # 消息ID
    group_id: str = "" # 群组ID，私聊为空
    sender: MessageMember  # 发送者信息
    message: List[BaseMessageComponent]  # 消息链
    message_str: str  # 纯文本消息内容
    raw_message: object # 平台适配器原始消息对象
    timestamp: int  # 消息时间戳
```

#### 3.1.3 消息链与消息段
消息链是一个有序列表`List[BaseMessageComponent]`，每个元素为一个消息段，描述消息的组成部分。
通用消息段类型：
| 消息段类型 | 作用 |
|------------|------|
| `Plain` | 纯文本消息段 |
| `At` | @提及消息段 |
| `Image` | 图片消息段 |
| `Record` | 语音消息段 |
| `Video` | 视频消息段 |
| `File` | 文件消息段 |

OneBot v11平台（QQ）额外支持的消息段：
`Face`(表情)、`Node`/`Nodes`(合并转发)、`Poke`(戳一戳)

### 3.2 指令系统
#### 3.2.1 基础指令
使用`@filter.command("指令名")`注册，用户发送`/指令名`即可触发：
```python
@filter.command("hello")
async def hello_handler(self, event: AstrMessageEvent):
    '''基础指令示例'''
    yield event.plain_result("bonjour!")
```
> 注意：指令名不能带空格，否则会被解析为指令参数。

#### 3.2.2 带参指令
AstrBot会自动解析指令参数，并支持类型注解校验：
```python
@filter.command("add")
async def add(self, event: AstrMessageEvent, a: int, b: int):
    # 用户发送 /add 1 2 ，自动解析a=1，b=2
    yield event.plain_result(f"计算结果: {a + b}!")
```

#### 3.2.3 指令组
用于组织层级化指令，支持无限嵌套：
```python
# 注册一级指令组 /math
@filter.command_group("math")
def math(self):
    pass

# 注册二级子指令 /math add
@math.command("add")
async def add(self, event: AstrMessageEvent, a: int, b: int):
    yield event.plain_result(f"结果是: {a + b}")

# 注册二级子指令 /math sub
@math.command("sub")
async def sub(self, event: AstrMessageEvent, a: int, b: int):
    yield event.plain_result(f"结果是: {a - b}")

# 嵌套指令组示例：/math calc add
@math.group("calc")
def calc(self):
    pass

@calc.command("add")
async def calc_add(self, event: AstrMessageEvent, a: int, b: int):
    yield event.plain_result(f"计算结果: {a + b}")
```

#### 3.2.4 指令别名
v3.4.28及以上版本支持，为指令/指令组设置多个别名：
```python
@filter.command("help", alias={'帮助', 'helpme'})
async def help(self, event: AstrMessageEvent):
    # 发送 /help、/帮助、/helpme 均可触发
    yield event.plain_result("这是插件帮助文档")
```

### 3.3 事件过滤器
用于限制消息的触发条件，支持多过滤器组合（AND逻辑，全部满足才会触发）。

#### 3.3.1 消息类型过滤
```python
# 接收所有消息
@filter.event_message_type(filter.EventMessageType.ALL)
async def on_all_message(self, event: AstrMessageEvent):
    yield event.plain_result("收到了一条消息")

# 仅接收私聊消息
@filter.event_message_type(filter.EventMessageType.PRIVATE_MESSAGE)
async def on_private_message(self, event: AstrMessageEvent):
    yield event.plain_result("收到了一条私聊消息")

# 仅接收群聊消息
@filter.event_message_type(filter.EventMessageType.GROUP_MESSAGE)
async def on_group_message(self, event: AstrMessageEvent):
    yield event.plain_result("收到了一条群聊消息")
```

#### 3.3.2 消息平台过滤
```python
# 仅接收AIOCQHTTP和QQOFFICIAL平台的消息
@filter.platform_adapter_type(filter.PlatformAdapterType.AIOCQHTTP | filter.PlatformAdapterType.QQOFFICIAL)
async def on_qq_platform(self, event: AstrMessageEvent):
    yield event.plain_result("收到QQ平台消息")
```
支持的平台类型：`AIOCQHTTP`、`QQOFFICIAL`、`GEWECHAT`、`ALL`

#### 3.3.3 权限过滤
```python
# 仅管理员可触发的指令
@filter.permission_type(filter.PermissionType.ADMIN)
@filter.command("admin_test")
async def admin_test(self, event: AstrMessageEvent):
    yield event.plain_result("管理员指令触发成功")
```

#### 3.3.4 多过滤器组合
```python
# 仅私聊场景下，发送/helloworld才会触发
@filter.command("helloworld")
@filter.event_message_type(filter.EventMessageType.PRIVATE_MESSAGE)
async def private_helloworld(self, event: AstrMessageEvent):
    yield event.plain_result("你好，私聊用户！")
```

### 3.4 事件钩子
用于监听机器人生命周期、LLM请求、消息发送等关键节点，**事件钩子不能与指令/事件过滤器装饰器共用**。

#### 3.4.1 机器人生命周期钩子
```python
# AstrBot初始化完成时触发 v3.4.34+
@filter.on_astrbot_loaded()
async def on_astrbot_loaded(self):
    print("AstrBot 初始化完成")
```

#### 3.4.2 LLM相关钩子
```python
# 等待LLM请求锁时触发，适合发送等待提示
@filter.on_waiting_llm_request()
async def on_waiting_llm(self, event: AstrMessageEvent):
    # 此处不能使用yield发送消息，需用event.send()
    await event.send("🤔 正在思考中，请稍候...")

# LLM请求前触发，可修改请求参数
@filter.on_llm_request()
async def on_llm_req(self, event: AstrMessageEvent, req: ProviderRequest):
    # 自定义system_prompt
    req.system_prompt += "\n请使用简洁的语言回复"

# LLM请求完成后触发，可修改返回结果
@filter.on_llm_response()
async def on_llm_resp(self, event: AstrMessageEvent, resp: LLMResponse):
    print(f"LLM返回结果: {resp.completion_text}")
```

#### 3.4.3 消息发送钩子
```python
# 消息发送前触发，可装饰/修改消息内容
@filter.on_decorating_result()
async def on_decorating_result(self, event: AstrMessageEvent):
    result = event.get_result()
    # 在消息末尾添加感叹号
    result.chain.append(Plain("!"))

# 消息发送完成后触发
@filter.after_message_sent()
async def after_message_sent(self, event: AstrMessageEvent):
    print("消息发送成功")
```

### 3.5 优先级与事件传播
#### 3.5.1 执行优先级
指令、事件监听器、钩子均可设置`priority`参数，数值越大，执行优先级越高，默认优先级为`0`。
```python
# 高优先级指令
@filter.command("helloworld", priority=10)
async def high_priority_hello(self, event: AstrMessageEvent):
    yield event.plain_result("高优先级Hello!")
```

#### 3.5.2 事件传播控制
使用`event.stop_event()`可终止事件传播，后续所有插件、LLM请求等逻辑都不会执行。
```python
@filter.command("check")
async def check(self, event: AstrMessageEvent):
    is_valid = False # 自定义校验逻辑
    if not is_valid:
        yield event.plain_result("校验失败，终止处理")
        event.stop_event() # 停止事件传播
```

## 四、消息发送能力
AstrBot支持被动回复、主动推送两种消息发送模式，同时支持富媒体消息发送。

### 4.1 被动消息回复
被动回复指机器人接收到消息后，通过`yield`返回回复内容，支持多条消息依次发送：
```python
@filter.command("helloworld")
async def helloworld(self, event: AstrMessageEvent):
    # 依次发送两条纯文本消息
    yield event.plain_result("Hello!")
    yield event.plain_result("你好！")
    # 发送本地图片
    yield event.image_result("path/to/image.jpg")
    # 发送网络图片
    yield event.image_result("https://example.com/image.jpg")
```

### 4.2 主动消息推送
主动消息指机器人无需用户触发，主动向指定会话推送消息，适用于定时任务、通知推送等场景。
```python
from astrbot.api.event import MessageChain

@filter.command("save_session")
async def save_session(self, event: AstrMessageEvent):
    # 保存会话唯一标识
    self.user_session = event.unified_msg_origin
    yield event.plain_result("会话已保存，将主动推送消息")

# 定时任务中主动发送消息示例
async def push_message(self):
    if hasattr(self, "user_session"):
        # 构建消息链
        message_chain = MessageChain().message("这是主动推送的消息").file_image("path/to/image.jpg")
        # 发送消息
        await self.context.send_message(self.user_session, message_chain)
```

### 4.3 富媒体消息发送
使用`MessageChain`或消息段列表构建富媒体消息，支持多种媒体类型组合。
#### 4.3.1 基础富媒体消息
```python
import astrbot.api.message_components as Comp

@filter.command("rich_msg")
async def rich_msg(self, event: AstrMessageEvent):
    # 构建消息链，按顺序发送
    chain = [
        Comp.At(qq=event.get_sender_id()), # @发送者
        Comp.Plain("来看这个图："),
        Comp.Image.fromURL("https://example.com/image.jpg"), # 网络图片
        Comp.Image.fromFileSystem("path/to/local.jpg"), # 本地图片
        Comp.Plain("这是一张示例图片。")
    ]
    # 发送富媒体消息
    yield event.chain_result(chain)
```

#### 4.3.2 文件与语音消息
```python
import astrbot.api.message_components as Comp

# 发送文件
@filter.command("send_file")
async def send_file(self, event: AstrMessageEvent):
    chain = [Comp.File(file="path/to/file.txt", name="示例文件.txt")]
    yield event.chain_result(chain)

# 发送语音（仅支持wav格式）
@filter.command("send_audio")
async def send_audio(self, event: AstrMessageEvent):
    chain = [Comp.Record(file="path/to/audio.wav", url="path/to/audio.wav")]
    yield event.chain_result(chain)
```

#### 4.3.3 视频消息
```python
from astrbot.api.message_components import Video

@filter.command("send_video")
async def send_video(self, event: AstrMessageEvent):
    # 本地视频
    video = Video.fromFileSystem(path="test.mp4")
    # 网络视频（更通用）
    # video = Video.fromURL(url="https://example.com/video.mp4")
    yield event.chain_result([video])
```

#### 4.3.4 群合并转发消息
> 仅OneBot v11平台（QQ）支持
```python
from astrbot.api.message_components import Node, Plain, Image

@filter.command("send_forward")
async def send_forward(self, event: AstrMessageEvent):
    # 构建合并转发节点
    node = Node(
        uin=905617992, # 发送者QQ号
        name="机器人", # 发送者昵称
        content=[
            Plain("这是合并转发的第一条消息"),
            Image.fromFileSystem("test.jpg"),
            Plain("这是合并转发的第二条消息")
        ]
    )
    yield event.chain_result([node])
```

## 五、插件配置管理
AstrBot提供了可视化配置能力，用户可在WebUI直接修改插件配置，无需修改代码。

### 5.1 配置定义文件
在插件根目录创建`_conf_schema.json`文件，用于定义配置的结构、类型、默认值等信息。
#### 5.1.1 基础配置示例
```json
{
  "token": {
    "description": "API访问Token",
    "type": "string",
    "obvious_hint": true,
    "hint": "请在XX平台获取API Token"
  },
  "max_count": {
    "description": "最大调用次数",
    "type": "int",
    "default": 10,
    "hint": "每日最大调用次数限制"
  },
  "enable_feature": {
    "description": "是否启用新功能",
    "type": "bool",
    "default": false
  },
  "sub_config": {
    "description": "嵌套配置",
    "type": "object",
    "items": {
      "name": {
        "description": "名称",
        "type": "string"
      },
      "timeout": {
        "description": "超时时间",
        "type": "int",
        "default": 60
      }
    }
  }
}
```

#### 5.1.2 配置字段通用属性
| 属性名 | 作用 | 必填 |
|--------|------|------|
| `type` | 配置类型，支持`string`/`text`/`int`/`float`/`bool`/`object`/`list`/`dict`/`file`/`template_list` | 是 |
| `description` | 配置描述，显示在WebUI中 | 否 |
| `hint` | 配置提示信息，鼠标悬浮显示 | 否 |
| `obvious_hint` | 是否醒目显示提示 | 否 |
| `default` | 配置默认值 | 否 |
| `items` | `object`类型的子配置Schema | 是（object类型） |
| `invisible` | 是否在WebUI隐藏该配置，默认false | 否 |
| `options` | 下拉列表可选项，数组格式 | 否 |
| `editor_mode` | 是否启用代码编辑器模式，v3.5.10+支持 | 否 |
| `editor_language` | 代码编辑器语言，默认json | 否 |
| `_special` | 特殊选择器，v4.0.0+支持，可选`select_provider`/`select_persona`等 | 否 |

#### 5.1.3 特殊配置类型
##### 文件上传配置（v4.13.0+）
```json
{
  "demo_files": {
    "type": "file",
    "description": "示例文件上传",
    "default": [],
    "file_types": ["pdf", "docx"]
  }
}
```

##### dict类型配置

### 5.2 在插件中使用配置
AstrBot会自动解析配置Schema，并在插件实例化时将配置对象传入`__init__`方法。
```python
from astrbot.api import AstrBotConfig
from astrbot.api.star import Context, Star, register

@register("config_demo", "author", "配置示例插件", "1.0.0")
class ConfigDemoPlugin(Star):

    @filter.command("show_config")
    async def show_config(self, event: AstrMessageEvent):
        yield event.plain_result(f"当前配置：\n最大调用次数：{self.max_count}")

    @filter.command("update_config")

        # 保存配置到文件

        yield event.plain_result(f"最大调用次数已更新为：{count}")
```

### 6.2 LLM工具（Tool）定义与注册
Tool是大模型调用外部能力的核心方式，支持两种定义与注册方式。

#### 6.2.1 类方式定义Tool
```python
from pydantic import Field
from pydantic.dataclasses import dataclass
from astrbot.core.agent.run_context import ContextWrapper
from astrbot.core.agent.tool import FunctionTool, ToolExecResult
from astrbot.core.astr_agent_context import AstrAgentContext

# 定义天气查询工具
@dataclass
class WeatherTool(FunctionTool[AstrAgentContext]):
    name: str = "get_weather"  # 工具名称
    description: str = "获取指定城市的天气信息"  # 工具描述，LLM会根据描述决定是否调用
    parameters: dict = Field(
        default_factory=lambda: {
            "type": "object",
            "properties": {
                "city": {
                    "type": "string",
                    "description": "需要查询天气的城市名称",
                },
            },
            "required": ["city"],
        }
    )

    # 工具执行逻辑
    async def call(
        self, context: ContextWrapper[AstrAgentContext], **kwargs
    ) -> ToolExecResult:
        city = kwargs.get("city", "")
        # 调用天气API获取数据（示例）
        weather_data = f"{city}当前天气：晴，25℃"
        return weather_data
```

#### 6.2.2 注册Tool到AstrBot
在插件初始化时注册Tool，注册后用户与LLM对话时，LLM可自动调用该工具：
```python

        # tool_mgr.func_list.append(WeatherTool())
```

#### 6.2.3 装饰器方式快速定义Tool

### 6.3 Agent调用
Agent是system_prompt+tools+llm的结合体，可自动处理工具调用循环，实现复杂智能体逻辑。
```python
from astrbot.core.agent.tool import ToolSet

@filter.command("agent_chat")
async def agent_chat(self, event: AstrMessageEvent, query: str):
    umo = event.unified_msg_origin
    prov_id = await self.context.get_current_chat_provider_id(umo)
    # 调用Agent
    llm_resp = await self.context.tool_loop_agent(
        event=event,
        chat_provider_id=prov_id,
        prompt=query,
        system_prompt="你是一个智能助手，会调用工具查询天气信息",
        tools=ToolSet([WeatherTool()]), # 绑定工具
        max_steps=30, # 最大执行步骤
        tool_call_timeout=60, # 工具调用超时时间
    )
    yield event.plain_result(llm_resp.completion_text)
```

### 6.4 Multi-Agent多智能体
v4.5.7+支持，采用`agent-as-tool`模式实现多智能体协同，将子智能体作为主智能体的工具调用。
核心实现逻辑：
1. 定义子智能体，封装为FunctionTool
2. 定义任务分配工具，用于主智能体选择子智能体
3. 主智能体通过tool_loop_agent调用，自动分配任务给子智能体

## 九、会话控制能力
v3.4.36+支持，用于处理多轮对话场景（如成语接龙、问答游戏等），实现用户与机器人的连续交互。

### 9.1 基础使用示例
以成语接龙为例，展示会话控制器的完整用法：
```python
import astrbot.api.message_components as Comp
from astrbot.api.event import filter, AstrMessageEvent
from astrbot.api import logger
from astrbot.core.utils.session_waiter import (
    session_waiter,
    SessionController,
)

@filter.command("成语接龙")
async def idiom_game(self, event: AstrMessageEvent):
    '''成语接龙游戏'''
    try:
        yield event.plain_result("游戏开始！请发送一个四字成语~")

        # 注册会话控制器，timeout为超时时间（秒）
        @session_waiter(timeout=60, record_history_chains=False)
        async def idiom_waiter(controller: SessionController, event: AstrMessageEvent):
            # 获取用户输入的内容
            user_input = event.message_str.strip()

            # 用户主动退出
            if user_input == "退出":
                await event.send(event.plain_result("游戏已结束，欢迎下次游玩~"))
                controller.stop() # 停止会话
                return

            # 校验输入是否为四字成语
            if len(user_input) != 4:
                await event.send(event.plain_result("请输入四字成语哦~"))
                return # 不停止会话，继续等待用户输入

            # 游戏逻辑处理
            # 示例：固定回复，实际需替换为成语接龙逻辑
            reply = user_input[-1] + "马当先"
            await event.send(event.plain_result(reply))

            # 重置超时时间，继续会话
            controller.keep(timeout=60, reset_timeout=True)

        # 启动会话控制器
        try:
            await idiom_waiter(event)
        except TimeoutError:
            # 超时触发
            yield event.plain_result("游戏超时，已自动结束！")
        except Exception as e:
            yield event.plain_result(f"游戏出错：{str(e)}")
        finally:
            # 停止事件传播，避免其他逻辑处理
            event.stop_event()
    except Exception as e:
        logger.error(f"成语接龙游戏出错：{str(e)}")
```

### 9.2 SessionController核心方法
| 方法名 | 作用 | 参数说明 |
|--------|------|----------|
| `keep()` | 保持会话 | `timeout`：超时时间；`reset_timeout`：是否重置超时计时器 |
| `stop()` | 立即结束会话 | 无参数 |
| `get_history_chains()` | 获取历史消息链 | 需开启`record_history_chains=True` |

### 9.3 自定义会话ID算子
默认会话以发送者ID为标识，可自定义算子实现群聊共享会话（如群内所有用户参与同一个游戏）。
```python
from astrbot.core.utils.session_waiter import SessionFilter

# 自定义会话过滤器
class GroupSessionFilter(SessionFilter):
    def filter(self, event: AstrMessageEvent) -> str:
        # 群聊使用群ID作为会话标识，私聊使用用户标识
        return event.get_group_id() if event.get_group_id() else event.unified_msg_origin

# 启动会话时传入自定义过滤器
await idiom_waiter(event, session_filter=GroupSessionFilter())
```

## 十一、插件开发核心规范
为保证插件的稳定性、兼容性和可维护性，请严格遵守以下开发规范：
1. **功能测试**：插件发布前必须经过充分测试，确保无崩溃、无逻辑错误
2. **代码注释**：核心逻辑、复杂函数必须添加清晰的注释，Handler必须添加描述注释
3. **数据存储规范**：
   - 持久化数据必须存储在`data`目录下，禁止存储在插件自身目录，避免更新时被覆盖
   - 大文件必须存储在`data/plugin_data/插件名/`目录下
4. **错误处理**：必须添加完善的异常捕获机制，禁止插件因单个错误导致整体崩溃
5. **代码格式化**：提交代码前，必须使用`ruff`工具格式化代码
6. **网络请求规范**：禁止使用`requests`同步网络库，必须使用`aiohttp`、`httpx`等异步网络库
7. **功能复用规范**：若为现有插件扩增功能，优先向原仓库提交PR，而非重复开发新插件（原插件停止维护除外）
8. **异步规范**：所有IO操作（网络请求、文件读写、数据库操作）必须使用异步实现，避免阻塞机器人主线程
